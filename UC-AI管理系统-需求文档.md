## 一、项目概述

### 1.1 项目背景

溃疡性结肠炎（UC）是一种慢性炎症性肠病，患者需要长期进行症状追踪、用药管理、饮食控制。现有痛点包括：

- 缺乏专业的UC症状追踪工具
- 饮食与症状的关联难以量化分析
- 复诊时无法提供完整的症状趋势数据
- 用药疗效评估依赖主观感受

### 1.2 产品定位

**一句话描述**：基于AI的UC患者日常管理系统，通过数据追踪和智能分析，帮助患者实现精准自我管理。

**核心价值**：
- 📊 **数据驱动**：标准化记录排便、饮食、症状、用药数据
- 🤖 **AI智能**：自动归因分析、个性化饮食推荐、疗效评估
- 🏥 **医患协作**：自动生成复诊报告，提升诊疗效率

### 1.3 目标用户

| 用户类型 | 核心需求 |
|----------|----------|
| **UC患者** | 症状追踪、饮食管理、用药提醒、复诊准备 |
| **消化科医生** | 获取患者30天结构化数据、疗效评估报告 |

### 2.2 模块一：排便记录

**对应文档**：[[UC日常记录与症状关联模型.md#二、排便记录标准化模型]]

#### 功能描述

基于Bristol大便性状量表，标准化记录每次排便的性状、血量、黏液等信息。

#### 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| bristol_type | 1-7 | ✅ | Bristol大便性状量表 |
| color | 枚举 | ✅ | yellow/brown/dark_brown/black/red/bloody |
| consistency | 枚举 | ✅ | formed/soft/mushy/watery |
| blood.amount | 枚举 | ✅ | none/trace/coin_size/moderate/heavy |
| blood.location | 枚举 | - | on_surface/mixed/pure_blood |
| mucus.amount | 枚举 | - | none/small/moderate/large |
| urgency.level | 0-10 | - | 排便急迫感 |
| pain_before.intensity | 0-10 | - | 排便前腹痛 |
| pain_after.intensity | 0-10 | - | 排便后腹痛 |

#### UI要求

- Bristol量表可视化组件（7种图标+文字说明）
- 一键快速记录（预设常用组合）
- 血量选择器（图标化：无/微量/硬币/明显/大量）

#### 数据输出

```json
{
  "stool_record": {
    "timestamp": "ISO8601",
    "form": {"bristol_type": 4, "color": "yellow"},
    "blood": {"present": true, "amount": "coin_size"},
    "mucus": {"present": true, "amount": "small"},
    "urgency": {"level": 5},
    "pain_before": {"intensity": 3}
  }
}
```

### 2.7 模块六：饮食-症状归因分析

**对应文档**：[[饮食-症状归因分析模型.md]]

#### 核心功能

回答用户核心问题：**"到底是哪一顿饭引起的症状？"**

#### 算法逻辑

1. **时间窗口匹配**：找出症状出现前72小时内的所有饮食记录
2. **多维度评分**：
   - 时间接近度（权重0.30）
   - 历史反应率（权重0.25）
   - 剂量效应（权重0.20）
   - 排除度（权重0.15）
   - 症状特异性（权重0.10）
3. **贝叶斯更新**：计算后验概率 P(食物→症状)
4. **混杂因素调整**：扣除压力、睡眠、用药等因素影响

#### 输出示例

```json
{
  "symptom": "bloody_stool",
  "candidate_foods": [
    {
      "rank": 1,
      "food": "猪里脊泥",
      "meal": "2026-04-29 晚餐",
      "hours_before": 11.5,
      "attribution_score": 0.68,
      "confidence": "medium",
      "reason": "时间窗口内 + 历史触发率25%"
    }
  ],
  "ai_recommendation": "猪里脊泥减半至40g观察"
}
```

#### UI要求

- 饮食-症状关联热力图
- 归因分析结果展示（Top N候选食物）
- 置信度标识（高/中/低）

### 2.9 模块八：粪便钙卫蛋白追踪

**对应文档**：[[粪便钙卫蛋白追踪模型.md]]

#### 核心功能

追踪FC（粪便钙卫蛋白）数据，评估黏膜愈合进度。

#### FC参考值

| FC值 (µg/g) | 解读 | 临床行动 |
|:----------:|------|----------|
| < 50 | 🟢 缓解 | 维持治疗，可考虑减量 |
| 50-250 | 🟡 轻度活动 | 优化治疗，4-8周复查 |
| > 250 | 🔴 中重度活动 | 评估升级治疗 |
| > 500 | 🔴 重度活动 | 紧急评估 |

#### 趋势分析

- 持续改善：FC持续下降 → 继续当前方案
- 平台期：连续2-3次不变 → 考虑优化/升级治疗
- 恶化型：FC持续上升 → 立即复诊

#### UI要求

- FC趋势折线图（含目标线50 µg/g）
- 黏膜愈合概率显示
- 下次检测提醒

### 2.10 模块九：用药疗效评估

**对应文档**：[[用药疗效评估模型.md]]

#### 核心功能

基于Mayo评分系统，自动评估药物疗效。

#### Mayo评分计算

| 维度 | 0分 | 1分 | 2分 | 3分 |
|------|------|------|------|------|
| 排便次数 | 正常 | 1-2次↑ | 3-4次↑ | ≥5次↑ |
| 便血 | 无 | 微量 | 明显 | 大量 |
| 黏膜表现* | 正常 | 轻度 | 中度 | 重度 |

> *无内镜时，用FC值估算：<50→0分，50-250→1分，250-500→2分，>500→3分

#### 疗效判定标准

| 疗效类别 | 判定标准 |
|----------|----------|
| 临床缓解 | Mayo ≤2分，且无单项>1 |
| 临床应答 | Mayo较基线↓ ≥3分 且 ↓ ≥30% |
| 部分应答 | Mayo↓1-2分 或 ↓ <30% |
| 原发性无效 | Mayo无变化 或 ↑ |
| 继发性失应答 | 缓解后Mayo再次↑ ≥3分 |

#### 治疗调整决策树

```
Mayo 0-2（缓解）🟢 → 维持治疗
Mayo 3-5（轻度）🟡 → 优化当前剂量
Mayo 6-8（中度）🟠 → 升级治疗
Mayo 9-12（重度）🔴 → 立即评估住院
```

### 2.11 模块十：复诊报告自动生成

**对应文档**：[[复诊报告生成模型.md]]

#### 核心功能

基于过去30天数据，AI自动生成结构化复诊摘要。

#### 报告内容

1. **执行摘要**（200字以内）
   - 30天整体评估：好转/稳定/恶化
   - 关键变化：便次、便血、腹痛
   - 治疗建议：继续/调整/升级

2. **症状详细分析**
   - 排便趋势图表
   - 腹痛频率/强度变化
   - 肠外表现汇总

3. **饮食耐受性分析**
   - 安全食物清单
   - 谨慎食物清单
   - 过敏原清单

4. **用药依从性评估**
   - 口服药依从性
   - 栓剂使用技巧

5. **AI建议：5个要问医生的问题**

6. **下一步治疗建议**

#### 触发时机

| 触发方式 | 说明 |
|----------|------|
| 自动 | 距离上次复诊满30天 |
| 手动 | 用户主动请求 |
| 症状恶化 | 预警等级变为红色时 |

#### 输出格式

- Markdown格式（可编辑）
- PDF导出（打印/邮件给医生）

### 2.12 模块十一：预警系统

**对应文档**：[[UC日常记录与症状关联模型.md#八、症状-症状关联模型]]

#### 预警等级

| 等级 | 触发条件 | 行动 |
|:----:|----------|------|
| 🟢 绿色 | 便次≤3，无血或微量，Bristol 4-5型 | 维持方案 |

#### 预警触发

- 实时计算：每次记录后自动评估
- 推送通知：橙色/红色预警时推送
- 预警日志：记录历史预警事件

### 4.2 AI引擎配置

**设计理念**：采用 OpenAI 兼容的 API 标准，支持用户自由切换 AI 模型提供商。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `api_base_url` | API 请求地址 | `https://api.deepseek.com/v1` |
| `api_key` | 用户 API Key | （用户填写） |
| `model_name` | 模型名称 | `deepseek-chat` |
| `max_tokens` | 最大输出 Token | `4096` |
| `temperature` | 温度参数 | `0.7` |

**支持的 AI 提供商**（开箱即用）：

| 提供商 | API 地址 | 推荐模型 |
|--------|----------|----------|
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-turbo` |

## 五、用户场景

### 场景一：日常记录

**用户**：小王，28岁，UC患者

**流程**：
1. 早上排便后，打开App
2. 选择Bristol 4型，标记"微量血"
3. 记录腹痛评分3/10
4. 系统自动计算今日预警等级：🟡黄色
5. 耗时约30秒

### 场景二：饮食-症状归因

**用户**：小李，35岁，想知道昨天便血的原因

**流程**：
1. 查看归因分析页面
2. 系统显示：便血出现前72小时的饮食时间线
3. AI分析：猪里脊泥（晚餐）归因得分0.68，置信度中等
4. 建议：下次减半至40g观察
5. 用户采纳建议，下次记录时自动追踪

### 场景三：复诊准备

**用户**：小张，42岁，准备去复诊

**流程**：
1. 复诊前3天，收到App提醒
2. 点击"生成复诊报告"
3. 系统生成30天摘要：
   - 症状趋势图表
   - 饮食-症状关联分析
   - 用药依从性统计
   - AI建议的5个问题
4. 导出PDF，带给医生

### 场景四：每日饮食推荐

**用户**：小陈，30岁，不知道今天吃什么

**流程**：
1. 早上打开App，查看今日推荐
2. AI根据昨日症状（便次3，微量血）生成推荐：
   - 早餐：白粥300g + 鸡蛋50g + 橄榄油10mL
   - 午餐：烂米饭250g + 猪里脊泥40g（减量观察）+ 南瓜泥150g
   - 晚餐：...
3. 用户按推荐执行，记录实际进食
4. 系统根据反馈更新耐受库
